```CPP
#include<bits/stdc++.h> 
#define RET(x)                { cout<<(x)<<endl; return; }
 
using namespace std;

void solve_case(int tc) {
  int n;
  cin>>n;
  int cnt[2]={0};
  for(int i=0; i<n; i++) {
    int x;
    cin>>x;
    cnt[x-1]++;
  }
 
  if(cnt[0]%2)
    RET("NO");
 
  if(!cnt[0] and cnt[1]%2)
    RET("NO");
 
  RET("YES");
}
 
int main() {
  int t = 1;
  cin>>t;
 
  for(int i=1; i<=t; i++)
    solve_case(i);
 
  return 0;
}
```

- ### Approach

1. It is impossible to divide the candies equally if their sum is odd
2. Again, it is not possible to divide candies equally if there is an odd number of `2`-s and no `1`-s. At least two `1 gram` candies (or `2n`, where `n` is a positive integer) are required for dividing the candies equally
3. Other than that, it is always possible to divide the candies into two equal halves

## 4.7: PASSING REFERENCES TO OBJECTS
### Example that demonstrates the usefulness of passing an object by reference:

- ### Code
```CPP
#include <iostream>

using std::cin;
using std::cout;
using std::endl;

class samp {
	int n;

public:
	samp(int n) {
		this->n = n;
	}

	void change_n(int n) {
		this->n = n;
	}

	void show() {
		cout << n << endl;
	}
};

void modify2(samp s, int n) { s.change_n(n); }
void modify(samp& s, int n) { s.change_n(n); }

int main() {
	samp s(10);
	modify(s, 20);
	s.show();

	modify2(s, 100);
	s.show();

	return 0;
}
```

- ### Output

![Screenshot](/assets/lab8/book.7.png)

- ### Discussion
This program illustrates the difference between passing an object by reference and by value in C++. The function `modify()` takes the object by reference, so changing `n` inside the function directly updates the original object in `main()`. However, `modify2()` takes the object by value, which creates a copy, so any modification affects only that copy and not the original object. As a result, the first change persists, while the second one does not.


## 4.8: RETURNING REFERENCES
### Program that contains a function that returns a reference:

- ### Code
```CPP
#include <iostream>

using std::cin;
using std::cout;
using std::endl;

class demo {
	int n;

public:
	demo() {}
	demo(int n) {
		cout << "Constructor called!" << endl;
		this->n = n;
	}

	~demo() {
		cout << "Destructor called!" << endl;
	}

	void show() {
		cout << n << endl;
	}
};

demo& retref(int n) {
	static demo d = demo(10);
	return d;
}

int main() {
	demo d = retref(10);
	cout << "Main function started" << endl;
	d.show();

	return 0;
}
```

- ### Output

![Screenshot](/assets/lab8/book.8.png)

- ### Discussion
This program demonstrates returning a reference from a function using a `static` object. The function `retref()` returns a reference to a static demo object, which is safe because static objects persist for the entire program lifetime. In main, the returned reference is used to initialize a new object `d`, which invokes the copy constructor and creates a separate object. As a result, two destructors are called at program termination: one for the local object `d` in main, and one for the static object inside `retref()`
